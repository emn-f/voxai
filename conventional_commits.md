# Conventional Commits

* **feat**: Para um novo recurso (um "feature"). Isso vira "Adicionado" no seu changelog.

* **fix**: Para uma correção de bug. Isso vira "Corrigido" no seu changelog.
* **chore**: Mudanças que não afetam o usuário final (ex: atualizar scripts de build, instalar dependências, CI/CD).
* **docs**: Mudanças apenas na documentação (ex: README.md, PRIVACY_POLICY.md).
* **style**: Mudanças de formatação de código (ex: linting, ponto e vírgula, etc).
* **refactor**: Refatoração de código que não corrige bug nem adiciona feature.
* **test**: Adição ou correção de testes.
* **perf**: Mudança que melhora a performance.
